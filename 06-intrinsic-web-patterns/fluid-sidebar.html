The Fluid Sidebar (The Asymmetrical Split)

This pattern establishes a priority-based layout. The sidebar requests a specific ideal (ex. 30ch), while the main content is configured to be 'greedier', aggressively filling all remaining space. Wrapping occurs only when the content area no longer meets it own minimum readable width, providing a natural, component-driven transition.
